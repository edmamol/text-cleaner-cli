# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in text-cleaner-cli, please report it responsibly. We appreciate your effort to keep our project secure.

### How to Report

- **Do not** open a public GitHub issue for security vulnerabilities
- Instead, email security concerns to the project maintainers directly
- Include details about the vulnerability and steps to reproduce it if possible
- Allow reasonable time (14-30 days) for a fix before public disclosure

## Security Considerations

This is a beginner-friendly educational project. While we strive to keep the code safe:

### What This Project Does

- Accepts text input from files
- Performs text normalization and cleaning operations
- Outputs processed text to files or terminal

### What This Project Does NOT Do

- Handle sensitive personal data
- Process encrypted content
- Authenticate users
- Manage network connections

## Best Practices for Users

1. **Validate input** - Be cautious with untrusted text files
2. **Check output** - Review cleaned text before using in production
3. **File permissions** - Ensure appropriate file permissions on input/output files
4. **Keep updated** - Pull the latest version to get any security improvements

## Dependencies

This project uses minimal dependencies to keep it simple and secure. We regularly check for vulnerabilities in any third-party packages.

## Scope

As an educational project, security isn't the primary focus. If you need enterprise-grade security features, please consider more mature alternatives.

Thank you for helping keep text-cleaner-cli secure!
